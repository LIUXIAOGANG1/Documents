首先欢迎您选择百度钱包提供的支付接入服务。此目录的例子是JAVA代码版本的，您可以直接在Myeclipse打开运行，也可以把所有文件放在WEB服务器上应用的目录下，进行测试运行。
1)文件列表说明
|-----------------------src\baifubao\bfb_sdk_comm.java	(共通函数文件)
|-----------------------WebRoot\WEB-INF\web.xml	(商家属性常量文件，商家可以在此文件中修改商家的ID)
|-----------------------key.properties (商户密钥文件，考虑到安全性，密钥需要放在外网访问不到的目录里。)
|-----------------------index.html (测试运行访问首页)
|-----------------------pay_unlogin.html (即时到账支付接口（不要求登录百付宝）测试数据输入页面)
|-----------------------pay_unloginServlet.java (即时到账支付接口（不要求登录百付宝），调用bfb_sdk_comm生成百付宝即时到账支付接口URL的DEMO)
|-----------------------pay_needlogin.html (即时到账支付接口（要求登录百付宝）测试数据输入页面)
|-----------------------pay_needloginServlet.java (即时到账支付接口（要求登录百付宝），调用bfb_sdk_comm生成百付宝即时到账支付接口URL的DEMO)
|-----------------------pay_wap.html (移动端即时到账支付接口（要求登录百付宝）测试数据输入页面)
|-----------------------pay_wapServlet.java(移动端即时到账支付接口（要求登录百付宝），调用bfb_sdk_comm生成百付宝即时到账支付接口URL的DEMO)
|-----------------------query_order_by_no.html (按订单号查询支付结果接口测试数据输入页面)
|-----------------------query_order_by_noServlet.java (按订单号查询支付结果接口，调用bfb_sdk_comm方法通过百付宝订单号查询接口查询订单信息的DEMO)
|-----------------------return_urlServlet.java (支付结果通知接收页面，页面同步page_url通知，异步通知return_url都可以参考该页面)

2)商家需要在WebRoot\WEB-INF\web.xml文件中修改成自己的商家ID
注意：商家ID和密钥需要同时修改才有效

3)支付成功的返回URL请在下单时指定
例如："http://localhost/callback.java"; 此页面实现代码请参照returnServlet.java
商家正式运行时，必须把自己的服务器部署在公网上的服务器上，这样支付成功后百度钱包的服务器才能将支付结果及时返回给商家

4)百度钱包支持三种支付方式：
	1.余额支付（必须登录百付宝）
	2.网银支付（在百付宝页面上选择银行，可以不登录百付宝）
	3.银行网关支付（直接跳到银行的支付页面，无需登录百付宝）
可以通过设定提交参数中pay_type的值来进行调整。取值范围参见《百度钱包企业版商户系统接入指南V3》附录

5)提交订单后在百付宝页面如出现“[5004]：您提交的参数有误”的错误时是提交参数有误，请重点检查订单号、订单创建时间、订单过期时间

6)提交订单后在百付宝页面如出现“[5804]：您提交的付款请求，存在异常，请勿重复操作”的错误时可能是验签失败，请确认您的商户号和密钥是否正确等
详见《百度钱包企业版商户系统接入指南V3》中如下章节：6.4 签名字符串和中文编码

7)在接收到支付结果通知后，判断是否进行过业务逻辑处理，不要重复进行业务逻辑处理
详见src\baifubao\bfb_sdk_comm.java 文件中的query_order_state()函数
注意：query_order_state()函数必须商户自己实现
